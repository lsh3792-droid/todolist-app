### UX 주석

- 이메일 필드: type="email", RFC 5321 형식 클라이언트 측 1차 검증
- 비밀번호 필드: type="password", 마스킹 처리

### UX 주석

- 기본 카테고리(업무/개인/기타): [수정] 및 [삭제] 버튼 시각적 비활성화(disabled) 처리, 클릭 불가 (BR-3.3.1)
- 사용자 정의 카테고리: [수정] 클릭 시 해당 행이 인라인 편집 입력란으로 전환
- [+ 카테고리 추가] 클릭: 목록 하단에 인라인 입력 행 추가
- 카테고리명 입력 후 Enter 키로도 저장 가능
- 삭제 성공 시: ✓ "카테고리가 삭제되었습니다" 토스트 메시지
- 추가/수정 성공 시: ✓ "카테고리가 저장되었습니다" 토스트 메시지
- 메인 할일 목록에서 헤더의 [카테고리 관리] 링크로 이동, 뒤로가기 가능

### UX 주석

- 이메일 필드: 읽기 전용(readonly), 변경 불가임을 명확히 표시
- 이름 수정과 비밀번호 변경은 독립된 섹션으로 각각 저장 처리 (별도 [저장] 버튼)
- 비밀번호 확인 필드: 새 비밀번호와 실시간 일치 여부 검증
- 이름 저장 성공 시: 헤더의 "[홍길동님]" 표시 즉시 갱신
- 비밀번호 변경 성공 시: 기존 Access Token 유지 (세션 중단 없음)
- [회원 탈퇴] 버튼: 위험 영역임을 강조하는 빨간색 계열 스타일 권장
- 탈퇴 확인 다이얼로그: '탈퇴 확인' 버튼 신중한 스타일 적용 (비가역 작업임을 강조)
- 탈퇴 성공 후: Zustand 스토어 초기화 → 로그인/회원가입 화면으로 이동
- Access Token 만료 상태에서 저장 요청 시: 자동 재발급 후 요청 재시도, Refresh Token도 만료면 로그인 화면으로 이동
